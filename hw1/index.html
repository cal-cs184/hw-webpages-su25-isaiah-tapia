Task 4: Barycentric coordinates

The primary usage of barycentric coordinates is to determine whether a point is located within a triangle. Each point on a triangle can be uniquely identified with (where A,B, and C represent the different vertices of the triangle).

\((x,y)= αA + βB + γC \)
\( α + β + γ = 1 \)

This property lends itself to thinking about the alpha beta and gamma values as a weights, essentially determining the amount of a value being represented on a screen due to the point's position on the triangle. If for instance every corner of an equilateral triangle represented a different color then a pixel position in the middle of the triangle would have all its weights set to 1/3. The figure to the right shows this relationship well. The rgb values all blur into one another as the points traverse towards the center and they distill from one another as they approach any vertex. This is due to me using the weights in the code to determine what color I should draw the weights. This is why if our any of our weight's values are not greater than or equal to zero our point is not located within the triangle. In code our barycentric coordinates would look like this.
